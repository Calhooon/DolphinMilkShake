## P5 blocking bugs discovered

### 1. Cert template mis-issuance on batch tail (unresolved)

Parent wallet (3321) serves wrong cert template to the last 5-8 workers in any large preflight batch. Workers declare `scraping` capability; parent returns a cert with `synthesis` instead. Workers time out after 60s.

**Pattern**: last ~8 lanes in any staggered launch fail cert audit. Restart of MetaNet clears the cache temporarily but it re-contaminates after ~20 successful issuances.

**Workaround**: dropped demo target from 28 → 10 lanes so we stay well below the contamination threshold.

### 2. Embedded wallet isolation poisoned all agents (FIXED)

Every dolphin-milk agent was auto-spawning its own `bsv-wallet daemon` subprocess pointing at `~/.dolphin-milk/wallet.db` (149 MB accumulated state from prior sessions). The configured `DOLPHIN_MILK_WALLET_URL` was being overridden by `ensure_own_wallet_url()` in `src/main.rs` (lines 447–513):

> If `~/.dolphin-milk/wallet.db` exists AND its identity doesn't match the configured URL, dolphin-milk picks a free port, spawns its own `bsv-wallet daemon`, and rewrites `cfg.wallet.url` to point there.

Consequence: all 30 agents across 10 lanes shared one wallet DB. The funded fleet wallets (captain-bsky-en-2 at 2M untouched) were never used. Agents drained the shared DB, hit "insufficient funds" errors, wallet.db locking caused balance queries to return empty strings.

**Fix**: moved `~/.dolphin-milk/wallet.db{,-shm,-wal}` aside (preserved as `.isolation-poisoner-<TS>`). On relaunch, `ensure_own_wallet_url()` returned None (no local state to defend), agents honored `DOLPHIN_MILK_WALLET_URL`, fleet wallets were correctly targeted.

**Verified after fix**: `captain-bsky-en-2` dropped from 2M → 1,733,487 in first cycle (previously would've stayed at 2M untouched). Tx counter started ticking (+225 on-chain in a few min).

**Upstream fix needed**: `cluster.js` should set `DOLPHIN_MILK_DATA_DIR` per agent so each has its own isolated workspace instead of all sharing `~/.dolphin-milk`. Alternative: dolphin-milk should honor `DOLPHIN_MILK_WALLET_URL` unconditionally and skip the isolation auto-spawn when the env var is explicitly set.
